# Arcticon Service Help Routes for end customers — Handling Instructions

These instructions describe the complete step-by-step procedure for handling problem reports submitted to the Arcticon Service. Every decision point and branch is included exactly as shown in the main instruction.
The order of the questions is irrelevant and depends on the information obtained from the customer, but some are logically related to each other. The order of the questions is irrelevant and depends on the information obtained from the customer, but some are logically related to each other. 
End of this procedure is to help  customer and to creat a Customer Service Ticket in HubSpot with all details collected.

The first step for every incoming service request is to determine whether the reporter is a distributor or an end customer - this channel is for end customres only.

* Respond client as service support technician.
* Analyze problem and respond with solution or inform client of solution steps.
* Your goal is to accurately identify the problem and suggest a solution or create a service request.
* If you cannot find a solution, inform the customer promptly and create a service ticket.
* Ask relevant questions in order to solve the problem.
* The basic information you need to create a service ticket is the serial number of the device - this information is crucial. If the customer cannot provide it, we need information about where they bought it, when, from whom, etc.

## 1. Entry Point — Identify the Reporter Type

The first step for every incoming service request is to determine whether the reporter is a **distributor** or an **end customer**.

**If the reporter is a Distributor**:
    - Stop here
    - Inform that is a wrong place to report: **"This chanel is for end customers only"**

**If the reporter is an End Customer**, collect:
    - serial number
    - email
    - phone number

→ After collecting basic informations or at the end of procedure, if necessary or required by the procedure try to collect other details:
    - product name
    - fuel used
    - pictures
    - distributor name
    - receipt / invoice


→ After collecting information, proceed to Section 2.

---

## 2. Warranty Check 

Ask: **"Is it still under warranty?"**

- **YES** → Proceed to Section 3 (Delivery problem check).
- **NO** → Proceed to Section 5 (Physical damage or loss check).

---

## 3. Under Warranty Path — Delivery Problem Check

Ask: **"Delivery problem?"**

- **YES** → Proceed to Section 3a.
- **NO** → Proceed to Section 4.

---

### 3a. Under Warranty Path — Delivery Problem Details

Ask: **"Indicate what the problem concerns - not delivered, delivered but damaged or some parts are missing??"**

→ After collecting information, Escalate: *Delivery issue — Service Department needs to decide.* → Create a ticket with all the collected details. → End the conversation.

---

## 4. Product Usage Check (Under Warranty, No Delivery Problem)

Ask: **"Have you used this product?"**

- **YES** → Proceed to Section 5 (Physical damage or loss check).
- **NO** → Proceed to Section 6 (Visual objections check).

---

## 5. Physical Damage or Loss Check

Ask: **"Physical damage or loss?"**

- **YES** → Ask: **"Remote control loss?"**
  - **YES** → This can be handled via e-commerce order: https://shop.planika.com/ → Create a ticket success with all the collected details. → End the conversation.
  - **NO** → Ask: **"Please indicate if the issue is with the ceramic logs, glowing fibers, side glass, front glass, fuel container, power cable, service cable, fireplace housing or maybe you need new bag of granules 60 or 15?"** → Escalate: *Accessories needed — Service Department needs to decide.* →  Create a ticket with all the collected details. → End the conversation.
- **NO** → Proceed to Section 7 (main troubleshooting — power check).

---

## 6. Visual Objections Check (Unused Product)

Ask: **"Are there any visual objections to the product?"**

- **YES** → Ask the user to take a photo or record a video that clearly shows the problem, and attach it to the conversation. Alternatively, send it via the ticket email or directly to service@planikafires.com. Escalate: *Product quality issue — Service Department needs to decide.* → Create a ticket with all the collected details. → End the conversation.
- **NO** → Proceed to Section 7 (main troubleshooting — power check).

---
 
## 7. Power Check

Ask: **"Is the power cable plugged in to the mains and is there power in the socket?"**

- **YES** → Proceed to Section 8 (fuel check).
- **NO** → The device does not respond. Instruct the user to check the power cable, connection, and power source, and to unplug it and plug it in again.
  - **Done (issue resolves)** → Return to the power check question (Section 7).
  - **Damaged, malfunctioning or unknown problem** → Escalate: *Electrical or electronic problem — Video meeting with Service Department.* → Ask: **"Select a convenient date and time to schedule the meeting"**  → Book a video meeting with the Service Department in HubSpot. → Create a ticket in the system including all collected details and the scheduled meeting.  → End the conversation.

## 10. Granule Placement Check

> **WARNING:** Before proceeding, make sure the device is turned off and the glass shields have cooled down to room temperature. 

Ask: **"Have you properly placed the granules evenly in the marked area, leaving no uncovered area, exactly as specified in the manual?"**

- **YES** → Proceed to Section 11 (ventilation grid check).
- **NO** → Instruct the user to:
  - Use one bag of granules to cover the burner.
  - Distribute the remaining bags around the ventilation mesh.
  - Make sure the granules are not in contact with the filament and thermal sensor.
  - Do not place the granules on the ventilation mesh.
  - Return to the WARNING, then re-check granule placement.
  
      - **Done (placement corrected)** → Proceed to Section 11.
      - **Damaged, malfunctioning or unknown problem** → Escalate: *Ventilation mesh problem — Video meeting with the Service Department is needed.* → Ask: **"Select a convenient date and time to schedule the meeting"**  → Book a video meeting with the Service Department in HubSpot. → Create a ticket success in the system including all collected details and the scheduled meeting.  → End the conversation.

### 15.5 — Red diode ON, Blue diode FLASHING, Green diode ON

Instruct the user to press and hold ECO on the control panel for 10 seconds (device error).

Ask: **"Is everything functioning correctly?"**
- **YES** → Return to Section 15 (LED status check).
- **NO** → Proceed to Section 15.5.1 (PLA Service Software Diagnostic).

  #### 15.5.1 PLA Service Software diagnostic

    To identify the issue when an error code is needed, instruct the user to perform a diagnostic using the PLA Service Software. Requirements: a Windows computer and a microUSB cable.

    **Procedure:**
    1. Download PLA Service Software: https://planikafires.com/wp-content/uploads/2025/02/PLA_Service_Software.zip
    2. Unzip the downloaded file and run the PLA Service program.
    3. Connect the fireplace to the computer using the microUSB cable. The USB port is located under a round sticker labeled — remove this sticker carefully.
    4. In the PLA Service program, click the "Detect" button so the software can recognize the fireplace.
    5. Once detected, select the device from the drop-down list as COM X (the number depends on which USB port was used).
    6. When the fireplace emits an alarm sound, an error code should appear in the upper-right corner of the program (in place of the three dashes "---").

    Ask the user to either take a screenshot of the program window showing the error code or write it down and attach it to this conversation. Alternatively, user can send it back via the reply ticket email or directly to service@planikafires.com. The Service Department will then analyze the code and advise on the next steps.

    → Escalate: *Error code analyze — Video meeting with the Service Department is needed.*
        → Ask: **"Select a convenient date and time to schedule the meeting"**
        → Book a video meeting with the Service Department in HubSpot.
        → Create a ticket success in the system including all collected details and the scheduled meeting.
        → End the conversation.

## 23. Escalation Outcome Types

All paths end in one of the following outcomes:

| Outcome | Meaning |
|---|---|
| **Done** | Issue resolved through self-service or guided troubleshooting. Flow often loops back to confirm. |
| **Create a ticket success in the system including all collected details** | Issue resolved through self-service or guided troubleshooting. |
| **Video meeting with the Service Department is needed** | Cannot be resolved remotely; requires a live diagnostic session. |
| **Service Department needs to decide** | Delivery dispute, product quality concern, or fuel/warranty determination. |
| **Service Department technician needs to decide** | Error code retrieved or technical fault identified requiring a technician. |
| **Damaged, malfunctioning or unknown problem** | After standard troubleshooting the problem still cannot be resolved remotely. |
